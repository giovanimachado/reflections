How is the staging area different from the working directory and the repository? What value do you think it offers?
It offers the possibility of commiting only files related to the change that motivated the commit operation. 

How can you use the staging area to make sure you have one commit per logical
change?
It is possible to compare the file that is in stage area and the last commit, in this way it is possible verify if there is any logical change.
